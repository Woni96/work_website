ROS 2 ROV Control Review

제어 리뷰 문서를 HTML / CSS / JS 기반 대시보드로 재구성했습니다

`control_code_review.md`와 각 컨트롤러별 리뷰 문서를 한 페이지에서 읽을 수 있게 정리했습니다. 핵심 리스크, 설계 철학, 우선순위, 모듈별 해설을 빠르게 탐색할 수 있습니다.

Top Risk

수동/자동 입력 freshness 처리의 불일치로 인해 입력 dropout 시 제어 상태가 latch될 수 있습니다.

Architecture

상위 제어기 → `wrench_merger` → `allocator_node`로 이어지는 분산형 구조는 추적성과 실험성이 좋습니다.

Actionability

timeout 추가, sign 파라미터화, allocator 성격 문서화처럼 바로 실행 가능한 개선안이 명확합니다.

1. Summary

전체 리뷰 핵심

Overview

좋은 점

  • 자세 / 수심 / 위치 / 병합 / allocation 단계 분리가 잘 되어 있음
  • `Wrench` 기반 인터페이스가 깔끔해서 확장성이 좋음
  • 수평/수직 스러스터 그룹 분리가 실제 ROV 구성과 잘 맞음
  • yaw release capture 같은 운용감 개선 로직이 존재함

주의할 점

  • 입력 끊김 시 마지막 명령이 남는 stale-state 리스크
  • 제어기별 override / freshness 철학이 완전히 일치하지 않음
  • allocator가 최적화 기반이 아니라 heuristic saturation 성격이 강함
  • 좌표계 부호나 z-coupling 의도가 코드만 봐서는 충분히 드러나지 않음

역할과 제어법칙

IMU quaternion 기반으로 `roll / pitch / yaw`를 계산하고, roll/pitch stabilizer + yaw heading hold에 가까운 구조로 동작합니다.

  • roll/pitch: PID + rate damping
  • yaw: heading error + yaw rate damping
  • trim, large tilt disable, yaw hold 정책 포함

리뷰 포인트

수동 wrench freshness 미관리로 보호 로직이 의도보다 오래 남을 수 있습니다.

`code_review/controller_review/controller_review_attitude.md:1`

핵심 수정 포인트: manual timeout, stale state reset, yaw override 복귀 일관화.

역할과 제어법칙

수심 센서와 IMU를 이용해 `Fz`를 만드는 depth PID controller입니다.

  • `target_depth - current_depth` 오차 기반 PID
  • pilot depth-rate 해석 가능
  • sensor offset compensation 포함
  • slew-rate 제한과 upward limit 포함

리뷰 포인트

manual heave release 시 목표를 `current_depth + offset`으로 잡는 정책이 직관과 어긋날 수 있습니다.

`code_review/controller_review/controller_review_depth.md:1`

핵심 수정 포인트: release offset 기본값 재검토, 의도 문서화.

역할과 제어법칙

world frame에서 위치 오차를 계산한 뒤 body frame 힘으로 바꾸는 planar PD position hold 구조입니다.

  • DVL position 또는 velocity integration 지원
  • yaw motion-aware damping 추가
  • manual release target capture 정책 포함

역할과 병합 정책

수동 입력과 각 자동 제어기 출력을 축별 규칙으로 합치는 supervisory mixer입니다.

  • `Fx/Fy`: manual override 아니면 position 사용
  • `Fz`: depth state와 manual heave를 함께 고려
  • `Tx/Ty`: attitude 유지 우선
  • `Tz`: manual yaw override 아니면 yaw hold

리뷰 포인트

역할과 구조

`Wrench(Fx, Fy, Fz, Tx, Ty, Tz)`를 8개 스러스터 명령으로 바꾸는 `control allocation + actuator shaping` 노드입니다.

  • 1~4번 수평 그룹: `Fx`, `Fy`, `Tz`
  • 5~8번 수직 그룹: `Fz`, `Tx`, `Ty`
  • pseudo-inverse 기반 분리 계산
  • priority / gain / normalization 포함
