System Overview

현재 제어기는 여러 제어 모듈이 협력해서 최종 스러스터 명령을 만드는 구조입니다

이 페이지는 `code_review/code/*.py`에 들어 있는 실제 제어 코드를 기준으로, 전체 제어 흐름과 각 모듈의 역할을 이해하기 쉽게 정리한 소개 페이지입니다. 자세한 함수 단위 코드 리뷰는 별도의 상세 리뷰 페이지에서 볼 수 있게 연결해두었습니다.

함수 중심 코드 리뷰 보기 모듈 설명 바로가기

입력 계층

조종기 입력, IMU, DVL, depth sensor 같은 센서/조작 입력이 각 제어 모듈로 들어갑니다.

제어 계층

`depth`, `position`, `attitude` 제어기가 각각 자기 축의 목표를 계산합니다.

출력 계층

`wrench_merger`가 제어 출력을 합치고 `allocator_node`가 8개 thruster 명령으로 바꿉니다.

1. Control Structure

전체 제어 흐름

Pipeline

Manual Teleop

조종자가 주는 수동 입력을 생성합니다.

`/rov/wrench_manual`, `/cmd_attitude`, `/rov/armed`

Depth Controller

수심 목표와 수동 heave를 해석해 `Fz`를 만듭니다.

`/ctrl/depth_heave`, `/ctrl/depth_active`

Position Controller

DVL와 IMU로 XY 위치 유지용 힘을 계산합니다.

`/ctrl/position_force`, `/rov/position_estimate`

Attitude Controller

roll / pitch / yaw 토크를 계산합니다.

`/ctrl/attitude_torque`

Wrench Merger

수동 입력과 자동 제어 출력을 축별 규칙으로 합칩니다.

`/rov/wrench_cmd`

Allocator

최종 wrench를 8개 스러스터 명령으로 변환합니다.

`/thruster_cmd`

이 구조가 의미하는 것

현재 시스템은 하나의 펌웨어 안에 모든 제어가 들어있는 형태가 아니라, ROS 2 노드들이 각자 역할을 나눠서 협력하는 분산형 제어 구조입니다.

그래서 특정 축만 개별적으로 튜닝하거나, 모듈 단위로 디버깅하고 교체하기가 쉽습니다.

읽는 포인트

  • 수동 조작과 자동 제어는 서로 완전히 분리되지 않고 함께 동작함
  • 깊이, 위치, 자세는 각자 계산 후 마지막에 하나의 wrench로 합쳐짐
  • 최종적으로 allocator가 물리적인 thruster 출력 형태를 결정함

3. Modules

각 모듈이 하는 일

Modules

`attitude_controller.py`

IMU를 바탕으로 roll, pitch, yaw를 안정화하는 자세 제어 모듈입니다.

  • 현재 자세와 목표 자세의 차이를 계산함
  • yaw hold와 manual yaw override를 함께 다룸
  • translation 중 trim attitude를 유지하려는 정책이 포함됨

`depth_controller.py`

현재 수심과 목표 수심의 차이를 바탕으로 heave 방향 출력을 만드는 모듈입니다.

  • depth hold와 pilot depth-rate를 함께 지원함
  • manual heave release 시 새 목표 수심을 다시 잡음
  • IMU를 이용한 depth sensor offset compensation이 있음

`position_controller.py`

DVL와 IMU를 사용해 XY 위치를 유지하기 위한 힘을 계산하는 모듈입니다.

  • world frame에서 위치 오차를 계산함
  • 최종 출력은 body frame 힘으로 다시 회전함
  • manual XY 조작 후 현재 위치를 새 hold target으로 다시 잡을 수 있음

`wrench_merger.py`

수동 입력과 각 자동 제어기 출력을 하나의 최종 wrench로 합치는 supervisory 모듈입니다.

  • 축별로 manual과 auto의 우선순위를 다르게 적용함
  • depth active 상태에 따라 heave 해석이 달라짐
  • 최종 출력 전의 마지막 의사결정 계층 역할을 함

`allocator_node.py`

최종 wrench를 실제 8개 thruster 명령으로 바꾸는 output allocation 모듈입니다.

  • 수평 4개, 수직 4개 스러스터 그룹으로 분리해 계산함
  • pseudo-inverse 기반 분배와 보상 항을 함께 사용함
  • slew-rate, deadband, output scaling 같은 actuator-friendly shaping이 포함됨
