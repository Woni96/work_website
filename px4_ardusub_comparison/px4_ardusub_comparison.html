3. 기능 차이 상세

PX4 UUV / ArduSub와의 해석 차이

Deep Dive

제어 구조 차이

  • 현재 코드는 depth / position / attitude / merge / allocation이 모두 분리됩니다.
  • PX4는 내부 flight task + controller + control allocation으로 통합됩니다.
  • 현재 구조는 축별 enable / disable과 디버깅이 쉽고, PX4는 시스템 연동성이 더 강합니다.

모드 구조 차이

  • PX4는 `Manual`, `Stabilized`, `Acro`, `Altitude`, `Position` 중심입니다.
  • 현재 구조는 명확한 mode switch보다 `control_enabled`, override threshold, `depth_active` 조합으로 움직입니다.

Allocator 철학 차이

  • 현재 allocator는 수평/수직 thruster 분리와 후처리 정책이 강합니다.
  • PX4는 `CA_ROTOR*`, `CA_R_REV`, `CA_ROTOR_COUNT` 중심의 일반화된 control allocation입니다.
  • 즉 현재는 기체 최적화형, PX4는 재사용성 높은 일반 allocator에 가깝습니다.

안전/시스템 기능 차이

  • 현재는 `armed` gating 중심의 비교적 단순한 안전 구조입니다.
  • PX4는 prearm check, output state machine, failure handling, offboard / return이 강합니다.

가까운 부분

  • manual heave를 depth-rate request로 해석하는 감각
  • 수동 조종 시 자동 hold를 약화하거나 해제하는 구조
  • vectored ROV 조이스틱 운용 전제
  • position hold / depth hold / stabilized 개념 분리

다른 부분

  • ArduSub은 mode 중심, 현재 코드는 노드 enable 중심입니다.
  • ArduSub은 EKF / mission / failsafe가 내장되어 있습니다.
  • 현재 position controller는 DVL + PD 기반 XY hold에 집중합니다.
  • 현재 attitude controller는 heave / XY motion protection 같은 커스텀 보호 로직이 많습니다.
  • 현재 allocator는 ArduSub frame mixer보다 더 커스텀성이 강합니다.

4. 파라미터 세팅 차이

제어 영역별 대응 관계

Parameters
4.1 자세 제어
현재 코드 PX4: `UUV_ROLL_P`, `UUV_PITCH_P`, `UUV_YAW_P`, `MC_*` ArduSub: `ATC_ANG_*`, `ATC_RAT_*`

현재 코드는 축별 P/ID와 override 정책, 보호 로직이 한 노드 안에 강하게 결합되어 있고, PX4 / ArduSub은 angle loop / rate loop / saturation 분리가 더 분명합니다.

4.2 깊이 제어
현재 코드 PX4: `UUV_GAIN_Z_P`, `MPC_Z_*` ArduSub: `PSC_POSZ_P`, `PSC_VELZ_*`

현재 depth controller는 ArduSub와 매우 유사한 조종 감각을 목표로 하며, pilot heave → depth-rate 명령, stick release → 현재 depth hold 흐름을 가집니다.

4.3 위치 제어
현재 코드 PX4: `UUV_GAIN_X/Y_*`, `MPC_XY_*` ArduSub: `PSC_VELXY_*`

현재 위치 제어는 PD 기반 XY hold 중심이며, DVL position 유무에 따라 velocity integration fallback을 갖습니다.

4.4 스틱 / 조종 입력
현재 코드 PX4: `UUV_STICK_MODE`, `UUV_SGM_*`, `UUV_RGM_*` ArduSub: `JS_GAIN_*`, `JS_THR_GAIN`

현재 입력 계층은 ArduSub처럼 파일럿 운용성을 중시하지만, GCS 중심이 아니라 ROS 노드 파라미터로 직접 정의됩니다.

4.5 Thruster allocation / frame
현재 코드 PX4: `CA_ROTOR*`, `CA_R_REV` ArduSub: `FRAME_CONFIG`, `MOT_*`

현재 allocator는 frame geometry와 운용 정책이 섞여 있는 커스텀 allocator이고, PX4 / ArduSub은 상대적으로 더 일반화된 체계를 제공합니다.

6. 결론

세 줄 결론

Conclusion

PX4와 비교

현재 구조는 더 실험적이지만 더 유연한 ROS형 구조입니다. 시스템 통합, 상위 모드, 안전 기능은 PX4가 더 강합니다.

ArduSub와 비교

제어 철학과 운용 감각은 ArduSub에 더 가깝지만, 현재 구조는 mode / EKF / failsafe / mission을 ROS 노드 조합으로 부분 재현한 형태입니다.

한 줄 요약

운용 감각은 ArduSub에 가깝고, 파라미터 구조는 커스텀이며, 추력 배분은 PX4보다 전용기체 최적화형, 시스템 통합도는 더 낮습니다.

  • `manual_wrench_timeout_sec` 기반 입력 신선도 관리
  • `min_target_depth`, `max_target_depth` 기반 depth limit
  • `/rov/armed` 기반 arm / disarm
  • DVL 기반 position hold
  • GUI 기반 파라미터 조정
  • `output_scale`, `max_output`, `deadband` 기반 추력 제한

한 줄 결론

현재 제어기는 저수준 제어 커스터마이징은 강하지만, ArduSub가 갖는 상위 시스템 기능인 mode, failsafe, EKF, mission, GCS 체계는 많이 비어 있습니다.
